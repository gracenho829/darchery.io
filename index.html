Whack-a-Mole

üë©üèª‚Äçüè´¬†Project Explanation

Our ‚ÄúWhack-a-Mole‚Äù project is a scaled down version of the classic arcade game Whack-A-Mole. Green light sensors act as the mole, randomly lighting up on the board, indicating where the user must hit. These light sensors are attached to a touch capacitor sensor so that when the user hits the conducive tape clipped to the sensor, the capacitor will notify the pi of a direct hit. This will turn off the green light and add points to the total score.

Our Whack-a-Mole project also implemented a bonus system where 10 correct, consecutive hits will initiate a bonus session. During this session, points that the user receive are doubled before being added to the total score. We attached an LED light that keeps track of the amount of consecutive hits and when the bonus session is initiated.

The score is displayed on a server and is updated by using the MQTT server.

We used the Paho MQTT client in Javascript to connect our own server with web socket protocol to receive messages(scores) from broker raspberry pi(our game).

üéâ¬†Verplank Diagram

üß≠¬†State Diagram

There are 2 states for the game: Normal Time and Bonus Time

Normal Time: After the player start the game, each time the player hit a mole, the player get 10 points, and a Green LED will light up on the LED stick. If the player miss a mole, a Red LED will light up and the LEDs will reset. The order is from left to right.

Bonus Time: When User hit 10/10 moles, the system will get into the Bonus Time so that the rainbow effect will start and all points are x5 times. Bonus Time will not stop until the play make a wrong hit.

The system also has a web application that will show the player‚Äôs score and play background music. The web application is connect with the Raspberry Pi, and every time the player hit a mole, the application will update the score and display it.

    def color_gradient(LED_stick, r1, b1, g1, r2, g2, b2, LED_length):
        # Subtract 1 from LED_length because there is one less transition color
        # than length of LEDs
        # LED_length = LED_length - 1
        # Calculate the slope of the line between r/g/b1 and r/g/b2
        r_slope = (r2 - r1) / LED_length
        g_slope = (g2 - g1) / LED_length
        b_slope = (b2 - b1) / LED_length
        # Set the color for each pixel on your LED Stick
        for i in range(0, LED_length):
            # Evaluate the ith point on the line between r/g/b1 and r/g/b2
            r_value = r1 + r_slope * i
            g_value = g1 + g_slope * i
            b_value = b1 + b_slope * i
            # Set the pixel to the calculated color
            time.sleep(0.02)
            LED_stick.set_single_LED_color(i + 1, int(r_value), int(g_value), int(b_value))
            time.sleep(0.02)
